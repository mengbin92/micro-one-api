# 第三阶段：账务服务迁移方案

## 1. 目标

将账务相关功能从单体应用迁移到独立的 `billing-service`，实现配额管理的中心化和一致性保证。

## 2. 迁移范围

### 2.1 需要迁移的功能

1. **配额预扣（ReserveQuota）**
   - Relay 请求前预扣配额
   - 支持流式场景的预估预扣
   - 预扣有效期管理

2. **配额提交（CommitQuota）**
   - 请求成功后确认预扣
   - 实际消费与预扣差异处理

3. **配额释放（ReleaseQuota）**
   - 请求失败时释放预扣
   - 超时自动释放

4. **消费流水（Ledger）**
   - 记录所有配额变更
   - 关联请求 ID 和用户
   - 支持流水查询

5. **配额充值（TopUpQuota）**
   - 用户充值
   - 管理员手动调整
   - 充值记录

6. **兑换码（RedeemCode）**
   - 兑换码生成
   - 兑换码校验
   - 兑换码使用记录

7. **账户快照（GetAccountSnapshot）**
   - 获取用户当前配额
   - 获取用户分组倍率
   - 获取冻结配额（预扣中）

## 3. Billing Service 架构设计

### 3.1 服务职责

1. 唯一的配额变更入口
2. 保证配额操作的原子性和一致性
3. 提供账务审计能力
4. 支持多种配额来源（用户配额、Token 配额）

### 3.2 分层设计

## 4. API 设计

### 4.1 Proto 定义

```protobuf
syntax = "proto3";

package billing.v1;

import "google/protobuf/timestamp.proto";

option go_package = "micro-one-api/api/billing/v1;v1";

// 配额预扣请求
message ReserveQuotaRequest {
  string user_id = 1;
  string request_id = 2;          // 关联的请求 ID
  int64 estimated_tokens = 3;     // 预估 Token 数量
  string model = 4;               // 使用的模型
  string channel_id = 5;          // 渠道 ID
}

// 配额预扣响应
message ReserveQuotaResponse {
  bool success = 1;
  string reservation_id = 2;      // 预扣 ID
  int64 reserved_amount = 3;      // 实际预扣金额
  string error_message = 4;
}

// 配额提交请求
message CommitQuotaRequest {
  string reservation_id = 1;
  int64 actual_tokens = 2;        // 实际 Token 数量
  int64 actual_cost = 3;          // 实际消费金额
  bool success = 4;               // 请求是否成功
}

// 配额提交响应
message CommitQuotaResponse {
  bool success = 1;
  int64 committed_amount = 2;     // 实际扣费金额
  int64 refund_amount = 3;        // 退还金额（预扣 - 实际）
  string error_message = 4;
}

// 配额释放请求
message ReleaseQuotaRequest {
  string reservation_id = 1;
  string reason = 2;              // 释放原因
}

// 配额释放响应
message ReleaseQuotaResponse {
  bool success = 1;
  int64 released_amount = 2;      // 释放金额
  string error_message = 4;
}

// 账户快照
message AccountSnapshot {
  string user_id = 1;
  int64 quota = 2;                // 可用配额
  int64 used_quota = 3;           // 已使用配额
  int64 request_count = 4;        // 请求次数
  string group = 5;               // 用户分组
  double group_ratio = 6;         // 分组倍率
  int64 frozen_quota = 7;         // 冻结配额（预扣中）
}

// 获取账户快照请求
message GetAccountSnapshotRequest {
  string user_id = 1;
}

// 获取账户快照响应
message GetAccountSnapshotResponse {
  AccountSnapshot snapshot = 1;
}

// 充值请求
message TopUpQuotaRequest {
  string user_id = 1;
  int64 amount = 2;               // 充值金额
  string operator_id = 3;         // 操作人 ID
  string remark = 4;              // 备注
}

// 充值响应
message TopUpQuotaResponse {
  bool success = 1;
  int64 new_quota = 2;            // 充值后配额
  string error_message = 3;
}

// 兑换码信息
message RedeemCode {
  string code = 1;
  int64 amount = 2;               // 兑换金额
  int32 count = 3;                // 剩余次数
  bool enabled = 4;
  google.protobuf.Timestamp created_at = 5;
}

// 创建兑换码请求
message CreateRedeemCodeRequest {
  string code = 1;
  int64 amount = 2;
  int32 count = 3;
  string operator_id = 4;
}

// 创建兑换码响应
message CreateRedeemCodeResponse {
  bool success = 1;
  string error_message = 2;
}

// 使用兑换码请求
message RedeemCodeRequest {
  string user_id = 1;
  string code = 2;
}

// 使用兑换码响应
message RedeemCodeResponse {
  bool success = 1;
  int64 amount = 2;               // 兑换到的金额
  int64 new_quota = 3;            // 兑换后配额
  string error_message = 4;
}

// 流水记录
message LedgerEntry {
  string id = 1;
  string user_id = 2;
  int64 amount = 3;               // 变更金额（正数增加，负数减少）
  int64 balance_after = 4;        // 变更后余额
  string type = 5;                // 类型：consume/recharge/refund/redeem
  string reference_id = 6;        // 关联 ID（request_id / reservation_id）
  string remark = 7;
  google.protobuf.Timestamp created_at = 8;
}

// 查询流水请求
message ListLedgerRequest {
  string user_id = 1;
  int32 page = 2;
  int32 page_size = 3;
  google.protobuf.Timestamp start_time = 4;
  google.protobuf.Timestamp end_time = 5;
}

// 查询流水响应
message ListLedgerResponse {
  repeated LedgerEntry entries = 1;
  int64 total = 2;
}

// Billing Service
service BillingService {
  // 预扣配额
  rpc ReserveQuota(ReserveQuotaRequest) returns (ReserveQuotaResponse);

  // 提交配额
  rpc CommitQuota(CommitQuotaRequest) returns (CommitQuotaResponse);

  // 释放配额
  rpc ReleaseQuota(ReleaseQuotaRequest) returns (ReleaseQuotaResponse);

  // 获取账户快照
  rpc GetAccountSnapshot(GetAccountSnapshotRequest) returns (GetAccountSnapshotResponse);

  // 充值
  rpc TopUpQuota(TopUpQuotaRequest) returns (TopUpQuotaResponse);

  // 创建兑换码
  rpc CreateRedeemCode(CreateRedeemCodeRequest) returns (CreateRedeemCodeResponse);

  // 使用兑换码
  rpc RedeemCode(RedeemCodeRequest) returns (RedeemCodeResponse);

  // 查询流水
  rpc ListLedger(ListLedgerRequest) returns (ListLedgerResponse);
}
```

## 6. 核心业务逻辑

### 6.1 预扣配额流程

```
1. 检查用户可用配额（quota - frozen_quota）
2. 计算预扣金额（tokens * model_price * group_ratio）
3. 扣减可用配额，增加冻结配额
4. 创建预扣记录（status=reserved）
5. 返回 reservation_id
6. 设置过期时间（默认 5 分钟）
```

### 6.2 提交配额流程

```
1. 查询预扣记录
2. 检查预扣状态（必须是 reserved）
3. 计算实际扣费（actual_tokens * model_price * group_ratio）
4. 如果请求成功：
   a. 扣减配额（实际金额）
   b. 释放冻结配额（预扣金额）
   c. 退还差额（预扣 - 实际）
   d. 更新预扣状态为 committed
   e. 创建消费流水
5. 如果请求失败：
   a. 释放冻结配额（全部预扣金额）
   b. 更新预扣状态为 released
   c. 创建释放流水
```

### 6.3 释放配额流程

```
1. 查询预扣记录
2. 检查预扣状态（必须是 reserved）
3. 释放冻结配额
4. 更新预扣状态为 released
5. 创建释放流水
```

### 6.4 兑换码流程

```
1. 查询兑换码
2. 检查兑换码是否启用
3. 检查剩余次数
4. 扣减剩余次数
5. 增加用户配额
6. 创建兑换记录
7. 创建流水
```

### 6.5 预扣过期清理

```
后台定时任务：
1. 查询所有状态为 reserved 且已过期的预扣
2. 释放冻结配额
3. 更新状态为 expired
4. 创建释放流水
```

## 11. 风险与应对

### 11.1 数据一致性风险

**风险**：预扣和提交之间数据不一致

**应对**：
1. 使用数据库事务保证原子性
2. 添加定时对账任务
3. 实现幂等性设计

### 11.2 性能风险

**风险**：高并发场景下性能不足

**应对**：
1. 使用 Redis 缓存账户信息
2. 数据库连接池优化
3. 异步写入流水记录

### 11.3 分布式事务风险

**风险**：跨服务调用导致事务失败

**应对**：
1. 实现重试机制
2. 实现补偿机制
3. 添加监控告警
